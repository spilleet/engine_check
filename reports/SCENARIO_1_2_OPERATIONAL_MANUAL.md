# ✈️ 항공 가스터빈 예지보전 시스템: 시나리오 1 & 2 실무 기능 보고서 (Operational Report)

본 보고서는 예지보전(MRO) 대시보드 시스템 내에서 **시나리오 1**과 **시나리오 2**가 실질적으로 어떠한 업무를 수행하고, 현장 정비사와 경영진에게 어떠한 실무적 가치를 제공하는지 사용자 시나리오 흐름 관점에서 설명합니다.

---

## 1. 시나리오 1: 실시간 지능형 알람 필터링 및 상황 전파

### 1.1 실질적으로 하는 일 (What it practically does)
100대의 항공기 엔진이 실시간으로 가동되는 모니터링 환경에서 **소음과 같은 무분별한 경보를 걸러내고, 진짜 위기(Danger) 상황이 닥쳤을 때 고장의 원인이 된 센서를 실시간으로 한글로 해석하여 비상 연락망에 즉각 전파**합니다.

```
[ 100대 엔진 실시간 가동 ] 
       │ (매 초 텔레메트리 센서값 수집)
       ▼
[ 이상 상태(Danger) 엔진 발생 ] ── (정상/점검 상태의 단순 경보는 필터링하여 노이즈 제거)
       │
       ▼
[ AlertFilteringAgent 기동 ] 
       │ 1. 21개 센서 중 이상치(Z-score 2.0 초과)를 보이는 범인 센서(예: s_11, s_12) 자동 식별
       │ 2. 고장 진단 내역을 인간이 읽기 쉬운 한국어로 실시간 자연어 요약
       ▼
[ 다채널 비상 상황 전파 ]
       • 슬랙(Slack) 채널 포맷의 텍스트 메시지 즉시 팝업
       • 심야 정비사 호출용 비상 음성 전화(TTS) 스크립트 실시간 생성 및 화면 표시
```

### 1.2 사용자 관점의 실제 동작 시나리오
1. 운영자가 대시보드에서 `실시간 스트림 시작` 버튼을 누르면 엔진 상태 그리드가 실시간으로 업데이트됩니다.
2. 특정 엔진(예: Unit #45)의 잔존수명(RUL)이 급격히 낮아져 위험 임계치(Danger, RUL 20 미만)에 도달하는 순간, 시스템이 이를 포착합니다.
3. 에이전트가 가동되어 **"이 엔진은 현재 연소기 온도 센서(s_11)가 기준치보다 2.3배 이상 높게 튀고 있어 연소실 내부 열 변형이 의심되는 심각한 상황입니다."**라는 한글 비상 메시지를 텍스트와 음성 대본(TTS)으로 대시보드 하단 알림창에 즉각 띄웁니다.
4. **실무적 가치**: 한밤중에 당직을 서는 정비사가 21개의 복잡한 센서 차트를 일일이 분석하지 않고도, 알람 창에 뜨는 2줄의 한글 브리핑만 읽고 즉각 부품 창고에서 어떤 도구를 챙겨 현장으로 출동해야 하는지 바로 알 수 있습니다.

## 2. 시나리오 2: 다중 에이전트 협동 기반 정비 작업지시서(Work Order) 자동 결재

### 2.1 실질적으로 하는 일 (What it practically does)
엔진 정비가 필요하여 결재를 올릴 때, **현장 정비사의 안전을 지킬 상세 수칙(LOTO)과 경영진을 설득할 정비 ROI(525%) 수치를 포함한 최고 품질의 정식 '정비 작업지시서'를 여러 LLM 에이전트가 토론하고 검수하여 자동으로 작성**해 줍니다.

```
[ 정비 필요 엔진 감지 (결재 대기) ]
       │
       ▼ [정비 승인/상신 클릭]
       │
┌──────┴──────────────────────────────────────────────────────┐
│  MaintenanceReportAgent (3대 협업 및 자가 교정 루프)         │
│                                                            │
│  [에이전트 1: 기술 서기] ──> 기술 분석 및 필수 LOTO 안전 지침 작성│
│                                  │                         │
│  [에이전트 2: 재무 플래너] ─> 정비 ROI(525%) 등 재무적 소명서 작성 │
│                                  │                         │
│  [에이전트 3: 품질 에디터] ─> 두 초안 병합 및 필수 요소 누락 검사 │
│            │ (LOTO 누락? 센서명 누락? 재무수치 오류?)           │
│            ├──> [반려 (REJECT)] ──> 피드백 안고 다시 작성(최대 3회)│
│            └──> [통과 (PASS)]                               │
└──────┬──────────────────────────────────────────────────────┘
       │ (백엔드 파이썬 코드 기반 최종 팩트체크 결합)
       ▼
[ 완벽한 정비 작업지시서(마크다운 파일) 최종 발행 ]
       • reports/submitted_orders/ 디렉토리에 파일 저장
       • 대시보드 모달 창에 고품질 문서 즉각 팝업 (엔지니어 즉시 출력 및 인쇄 가능)
```

### 2.2 사용자 관점의 실제 동작 시나리오
1. 위험군에 속한 엔진은 대시보드 우측 리스트에 **[결재 대기 (Pending Supervisor)]** 상태로 표시됩니다.
2. 상급자 모드에서 정비사가 **[결재 승인]** 버튼을 누릅니다.
3. 그 즉시 백엔드에서 3대의 에이전트가 협동 루프를 돕니다:
   - **기술 서기**가 센서 상태를 바탕으로 1차 초안을 적고, 고압 배관 작업이 있으므로 전기 및 밸브를 물리적으로 잠그는 **LOTO(Lockout-Tagout) 절차**를 명시합니다.
   - **재무 플래너**가 "이번 정비에 드는 비용은 \$8,000이지만, 정비를 안 해서 엔진이 불시에 멈춰 비행이 결항될 경우 벌어지는 AOG 패널티 벌금 \$50,000를 막아주므로 **방어 순이익 \$42,000 및 ROI 525%**의 경영 효율을 달성합니다"라는 수치 분석 단락을 덧붙입니다.
   - **품질 에디터**가 이 둘을 취합하여 엄밀하게 심사합니다. 만약 "LOTO"라는 중요 안전 단어나 해당 엔진의 구체적인 "이상 센서명"이 누락되었다면 가차 없이 **반려(REJECT)**를 치고 "정비사의 목숨이 위험하니 LOTO 지침을 보강하고 타깃 센서를 명시하시오"라는 피드백과 함께 돌려보냅니다.
   - 에이전트들은 지적받은 내용을 보강해 문서를 고쳐 씁니다. 통과되면 마침내 **PASS**를 획득합니다.
4. 최종 결재가 완료되면 대시보드 화면에 고급 마크다운으로 포맷팅된 정식 작업지시서 팝업창이 뜨며, 실제 `reports/submitted_orders/` 폴더 내에 마크다운 문서로 안전하게 저장됩니다.
5. **실무적 가치**: 기존에 엔지니어들이 서류 작성에 낭비하던 시간을 제로(0)로 단축해 주며, 안전 수칙 누락으로 인한 중대재해 처벌법 위반 위험이나 근거 없는 비용 책정으로 인한 예산 거절 문제를 시스템 레벨에서 원천 예방합니다.

---

## 3. 실무 시나리오의 핵심 비즈니스 가치

* **안전 규정(LOTO) 100% 준수**: 사람이 작성할 때 깜빡할 수 있는 필수 안전 단어가 빠지면 결재 문서가 발행되지 않으므로 산업 안전사고를 기계적으로 차단합니다.
* **경영진 보고 문서 작성의 자동화**: 복잡한 기계 공학적 센서 지표를 비전문가 임원진도 이해할 수 있는 재무 지표(ROI 525%)로 자동 번역하여, 기업 내부 의사결정 속도를 비약적으로 단축시킵니다.
* **현장 정비 업무의 가속화**: 정비사가 고장을 감지하고, 결재를 받고, 작업지시서를 들고 작업장에 투입되기까지의 모든 행정 절차를 수 분 이내로 완전 자동화 및 간소화합니다.
